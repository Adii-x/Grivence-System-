# Nexus - Complete Project Guide

This document is a simple start-to-end guide for your project.
It explains what the application does, who uses it, and how to use each part.

---

## 1) What is Nexus?

Nexus is a **Student Grievance Management System**.
It helps students raise complaints and helps the college team track and resolve them in an organized way.

Instead of complaints being lost in chats or paper forms, everything is available in one dashboard.

---

## 2) Why this project is useful

- Students get a proper way to submit issues.
- Staff can manage and update complaints by department.
- Admin can monitor progress and check overall performance.
- Everyone can see updates and comment history in one place.

---

## 3) Who can use the system

There are 3 types of users:

1. **Student**
2. **Staff**
3. **Admin**

Each role has different screens and permissions.

---

## 4) Main features (simple view)

- User login and role-based dashboard
- Student complaint submission form
- Optional attachment upload with complaint
- Status tracking (Pending, In Progress, Resolved, Escalated)
- Department-wise assignment
- Staff comments and updates
- Admin-level complaint monitoring and insights

---

## 5) How the complaint flow works

1. Student creates a complaint.
2. Complaint is assigned to the relevant department.
3. Staff reviews it and updates status/deadline.
4. Staff and student can see comment updates.
5. Complaint gets resolved (or escalated if needed).
6. Admin can monitor all complaints and overall health.

---

## 6) Student module - what student can do

### Student can:

- Register and login
- Submit a new complaint
- Choose category and priority
- Add complaint details clearly
- Upload a supporting file/image (optional)
- Track all submitted complaints
- Open each complaint and check:
  - Current status
  - Timeline progress
  - Comments from staff
  - Last update date

### Good use practice for students:

- Write clear title and detailed description.
- Add proof (attachment) whenever possible.
- Check dashboard regularly for updates.

---

## 7) Staff module - what staff can do

### Staff can:

- Login to staff panel
- View complaints assigned to their department
- Open each complaint and see:
  - Student details (name, student ID, department, year)
  - Complaint details
  - Uploaded attachment (if student provided one)
- Update complaint status
- Set or update deadline
- Add comments for progress updates

### Why this helps staff:

- Better task tracking
- Easy communication with students
- Faster issue handling

---

## 8) Admin module - what admin can do

### Admin can:

- Login to admin dashboard
- View all complaints across departments
- Check complaint status and priorities
- Open complete complaint detail with student info
- Track department-level performance
- Identify delays, overdue cases, and bottlenecks

### Why this helps admin:

- Better visibility
- Better accountability
- Better decision making for institutional improvement

## 10) How to run the project (basic)

### Step 1: Install dependencies

- Install backend dependencies
- Install frontend dependencies

### Step 2: Add environment values

- Add database connection
- Add secret keys
- Add allowed frontend URL for CORS
- Add frontend API base URL

### Step 3: Start backend

- Run backend server

### Step 4: Start frontend

- Run frontend app and open it in browser

### Step 5: Use role accounts

- Login as student/staff/admin to test complete flow

## 11) Suggested demo script (for presentation)

Use this sequence to demonstrate project clearly:

1. Register/Login as Student
2. Submit complaint with attachment
3. Show student dashboard and complaint status
4. Login as Staff (different browser/incognito)
